# Docs Directory

## Ownership
This directory is owned by the Tech Lead.

## Files
- `tech-design.md` — Technical design document
- `architecture.md` — Architecture overview
- `qa/` — QA reports per task
- `devlog.md` — Chronological development timeline

## Rules
- Tech design must reference both PRD and design spec
- Architecture doc should include system diagrams
- QA reports are auto-generated by the build agent after each task
- All technical decisions should be documented with rationale
- The devlog is append-only — never delete or modify existing entries
- Developers are strongly encouraged to add learnings before merging PRs

## Tech Design Sections
1. Architecture Overview
2. Data Models
3. API Design
4. Third-Party Integrations
5. Security Considerations
6. Performance Requirements
7. Migration Strategy (if applicable)
