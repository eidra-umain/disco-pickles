# PM Directory

## Ownership
This directory is owned by the Product Manager.

## Files
- `prd.md` — Product Requirements Document
- `summary.md` — Project summary and context

## Rules
- PRD must include: problem statement, user stories, success metrics, scope, and timeline
- Summary should be concise enough for any new team member to understand the project in 2 minutes
- All changes to PRD require PM approval
- Use the `prd-generator` skill to create or update the PRD

## Templates
When creating a PRD, include these sections:
1. Problem Statement
2. Target Users
3. User Stories
4. Success Metrics
5. Scope (In/Out)
6. Timeline & Milestones
7. Open Questions
