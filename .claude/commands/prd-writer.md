# Product Requirements Document

Generates a comprehensive Product Requirements Document with 14 sections. Stakeholder-ready. Calibrated to project scope — from 3 pages for a small feature to 18 for a full product.

## Guidelines

- 14 sections: executive summary, user stories, functional requirements...
- Non-functional requirements (performance, security, accessibility)
- Success metrics with baselines and targets
- Every requirement prioritized: P0 (must have) → P1 → P2

## Prerequisites

docs/project-summary.md must exist

## Output

docs/PRD.md
