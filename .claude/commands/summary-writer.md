# Project Summary

Takes the messy brainstorm and distills it into a structured one-page summary. Extracts the what, why, who, scope, key features, and open questions.

## Guidelines

- Preserves your exact language and intent
- Flags anything it inferred (marked [inferred])
- Separates decisions made from open questions
- Must exist before /prd-writer can run

## Output

docs/project-summary.md
